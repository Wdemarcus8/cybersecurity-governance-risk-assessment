# Cybersecurity Governance, Risk & Compliance (GRC) Assessment
### Fictional Case Study: Meridian Pay Solutions

## Overview

This project is a comprehensive GRC assessment conducted for **Meridian Pay
Solutions**, a fictional mid-size fintech company (~250 employees) providing
payment processing and digital wallet services to small-to-mid-size merchants.

The assessment was performed using the **NIST Cybersecurity Framework (CSF) 2.0**
as the primary governance framework, with risk assessment methodology informed
by **NIST SP 800-30**. The goal is to evaluate the organization's current
cybersecurity posture, identify gaps against the framework, translate those
gaps into a prioritized risk register, and produce an actionable remediation
plan (POA&M).

> **Note:** Meridian Pay Solutions is a fictional company created for portfolio
> demonstration purposes. All data, findings, and artifacts in this repository
> are illustrative and do not represent a real organization.

## Why This Project

This project demonstrates practical GRC analyst skills, including:
- Applying the NIST CSF 2.0 framework to assess organizational security posture
- Conducting asset inventory and classification
- Performing a gap assessment against a recognized framework
- Translating qualitative findings into a quantitative, prioritized risk register
- Developing a Plan of Action & Milestones (POA&M) for remediation tracking
- Communicating findings to both technical and executive audiences

# Cybersecurity Governance Risk Assessment
This project demonstrates the application of Governance, Risk, and Compliance (GRC) principles through a simulated cybersecurity assessment of a fictional financial services organization, Acme Financial Services. The assessment was conducted using concepts from the NIST Cybersecurity Framework (CSF) and common cybersecurity risk management practices.

The objective of this project was to identify organizational assets, evaluate cybersecurity risks, assess security control gaps, document remediation efforts, and provide executive-level recommendations to improve the organization's security posture.

-Conduct a cybersecurity gap assessment.
-Develop a Plan of Action and Milestones (POA&M).
-Provide executive-level recommendations for remediation and risk reduction.
-Project Deliverables

**Asset Inventory**

An inventory of critical organizational assets used to support business operations and security governance activities.

**Risk Register**

A documented list of identified cybersecurity risks including likelihood, impact, risk scores, and recommended treatment strategies.


**NIST CSF Gap Assessment**

An evaluation of the organization's current cybersecurity maturity against the five functions of the NIST Cybersecurity Framework:

-Identify
-Protect
-Detect
-Respond
-Recover
-Plan of Action & Milestones (POA&M)

A remediation tracking document is used to manage security findings, assign ownership, and monitor corrective actions.


**Executive Summary**

A management-level report summarizing assessment findings, key risks, and strategic recommendations.


**Key Risks Identified**

The assessment identified several areas requiring improvement:

-Phishing and social engineering attacks
-Weak password management practices
-Unpatched systems and software
-Excessive user privileges
-Ransomware exposure
-Third-party vendor risk
-Data leakage incidents
-Lost or stolen company devices
-Insider threats
-Backup and recovery failures
